# 多元拾光 UI 设计规范

版本：1.0  
基准：首页沉浸式视觉系统  
适用范围：社区首页、AIGC、模型广场、闪念、教程、活动、会员、积分、消息、用户中心及详情/投稿页面

## 1. 设计方向

多元拾光采用“沉浸式暗场创作空间”作为统一视觉语言。界面以近黑背景承载内容，以暖金作为主要动作与选中反馈，以酸橙色表达邀请增长场景，以白色表达中性商城与信息入口。组件强调胶囊轮廓、低对比玻璃表面、克制流光和线型图标，避免高饱和大面积渐变与厚重描边。

关键词：

- 沉浸、克制、电影感
- 深色玻璃、暖金高光
- 胶囊控件、线型图标
- 清晰层级、轻量动效

## 2. 设计令牌

### 2.1 色彩

| 令牌 | 值 | 用途 |
| --- | --- | --- |
| `--ds-bg` | `#05070b` | 页面背景 |
| `--ds-bg-raised` | `#0b0f13` | 抬升区域 |
| `--ds-surface` | `rgba(11, 15, 19, .74)` | 卡片与控件 |
| `--ds-surface-strong` | `rgba(20, 22, 24, .92)` | 弹层、展开菜单 |
| `--ds-text` | `#fffaf0` | 主文字 |
| `--ds-text-muted` | `rgba(255, 250, 240, .62)` | 次级文字 |
| `--ds-text-faint` | `rgba(255, 250, 240, .42)` | 辅助信息 |
| `--ds-line` | `rgba(255, 247, 230, .14)` | 轻描边 |
| `--ds-gold` | `#f4c95d` | 主动作、选中态 |
| `--ds-gold-soft` | `rgba(244, 201, 93, .14)` | 金色浅底 |
| `--ds-lime` | `#d9ff8c` | 邀请增长入口 |
| `--ds-white` | `#ffffff` | AI 商城及中性高亮 |
| `--ds-danger` | `#ef664e` | 危险状态 |
| `--ds-success` | `#68ca93` | 成功状态 |

### 2.2 字体

- 品牌展示：`Montserrat Local`，仅用于英文主标题和品牌型数字。
- 中文正文：系统中文字体栈，保证跨平台清晰度。
- 主标题：`600–700`。
- 组件文字：默认 `400`，选中态 `520–600`。
- 辅助文字：`200–400`，不使用过粗字重。
- 数字：启用等宽数字特性，积分与价格保持稳定。

### 2.3 间距与圆角

- 基础间距单位：`4px`。
- 常用间距：`8 / 12 / 16 / 20 / 24 / 32 / 48px`。
- 小控件圆角：`10px`。
- 卡片圆角：`16px`。
- 弹层圆角：`20px`。
- 导航、Tab、按钮、选择器：`999px` 胶囊。

### 2.4 阴影与模糊

- 控件内高光：`inset 0 1px 0 rgba(255,255,255,.08)`。
- 普通浮层：`0 12px 32px rgba(0,0,0,.24)`。
- 强浮层：`0 24px 64px rgba(0,0,0,.42)`。
- 玻璃模糊：`blur(16px) saturate(1.12)`。
- 金色发光只用于主动作与选中态，透明度不超过 `.18`。

## 3. 导航规范

- 顶部导航高度：`76px`。
- Logo 使用组合版 `LOGO.svg`，不再分离图形与文字。
- 一级导航默认高度 `36px`，左右内边距 `20px`。
- 当前一级导航高度 `42px`，使用低透明黑色胶囊、轻描边和线型图标。
- 一级导航间距：`4–8px`。
- 右侧动作全部使用胶囊：
  - 活动中心：暖金强调。
  - 邀请有礼：酸橙色 `#d9ff8c`。
  - AI 商城：白色文字与中性玻璃底。
  - 消息：独立圆形图标按钮。
  - 积分与头像：合并为一个账户胶囊。
- 图标统一使用 Remix Icon 的线型版本，默认 `16px`。

## 4. 按钮规范

### 主按钮

- 高度：`42–48px`。
- 圆角：胶囊。
- 背景：暖金低透明渐变叠加深色底。
- 文字：暖金，字重 `520`。
- Hover：上移 `1px`、描边增强、出现轻微金色外发光。

### 次按钮

- 深色玻璃底、轻描边、白色文字。
- Hover 只增强背景和描边，不改变布局。

### 文字按钮

- 无背景、无描边。
- 使用文字颜色和右箭头表达下一步。

### 积分动作

- 积分数值前使用 `copper-coin-line`。
- 生成类按钮不再配置额外左侧业务图标。

## 5. Tab 规范

- Tab 容器使用深色玻璃胶囊，内边距 `4px`。
- Tab 高度 `40–44px`，最小水平内边距 `16px`。
- 默认文字使用次级色，字重 `400`。
- 选中项使用暖金浅底、暖金文字、轻描边与内高光。
- Hover 不改变容器尺寸，不移动分割线。
- 图标可选；使用时统一为 Remix Icon 线型 `16–18px`。

## 6. 表单与选择器

- 输入框和选择器高度 `48–52px`。
- 使用深色玻璃底与 `--ds-line` 描边。
- 默认文字字重 `400`；Placeholder 字重 `200`、透明度约 `.52`。
- Focus 使用暖金描边和轻微外发光，不改变尺寸。
- 下拉菜单向可用空间方向展开，菜单与触发器左侧对齐。
- 选择器宽度由当前内容决定；长内容最大宽度受容器限制并省略。

## 7. 卡片规范

- 卡片以背景层级和阴影区分，不使用大面积高对比描边。
- 默认圆角 `16px`，内容区最小内边距 `20px`。
- Hover 上移 `2px`，媒体轻微放大，阴影增强。
- 图片与视频使用相同的媒体遮罩和底部黑色渐变，保证文字可读。
- 状态、标签、模型信息使用小型胶囊，不使用彩色实心矩形。

## 8. 动效规范

- 快速反馈：`160ms ease`。
- 控件状态切换：`220ms ease`。
- 弹层与内容进入：`420ms cubic-bezier(.16,1,.3,1)`。
- 列表进入使用轻微位移与透明度，不使用模糊入场。
- 所有位移动效必须支持 `prefers-reduced-motion`。

## 9. 响应式规范

- `1366–1920px`：完整导航与账户胶囊，内容容器最大宽度 `1280px`。
- `< 1366px`：缩小导航间距和右侧按钮水平内边距，不减少导航项目。
- `< 1100px`：辅助文案可隐藏，关键动作保留。
- `< 900px`：导航允许横向滚动或收拢，表单与按钮改为单列。
- `< 640px`：卡片单列、弹层使用视口宽度、主要按钮占满容器。

## 10. 无障碍

- 所有图标按钮必须提供 `aria-label`。
- 文字与背景对比度应满足 WCAG AA。
- Focus 可见状态不得仅依赖颜色变化。
- 交互目标最小尺寸 `40×40px`。
- 动效不能成为理解或完成任务的唯一方式。
